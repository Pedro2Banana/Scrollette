# Scrollette 核心流程说明

用简单流程描述三件事：打开一本书、阅读过程、对话时各自发生了什么。

涉及的两类存储，先记住它们职责不同：
- **SQLite（`data/scrollette.db`）**：阅读元数据——哪本书、哪些页、什么时候读的、上次停在哪。
- **向量库 Chroma（`data/rag_index/`）**：已读页切成的 chunk + 向量，供检索。
- 另外还有**内存里的对话记忆**（`Conversation`）：只存最近几轮的「问题 + 回答」。

---

## 一、打开一本书

```
打开 PDF
  ---> 计算 file_hash（作为 document_id，文件挪位置也认得出同一本书）
  ---> documents 表登记本书（upsert_document：文件名、页数、打开时间）
  ---> 从库恢复 max_read_page（companion 检索的“只到已读页”范围）
  ---> 从库恢复上次视图（last_page 上次停留页、two_page 单/双页）
  ---> 跳到该页、设好单/双页、渲染
```

注意：**打开时不全量索引整本书**（那样又慢又贵）。索引是“边读边来”的（见下）。

---

## 二、阅读过程中

```
翻页 / 跳页 / 切单双页
  ---> 渲染当前页
  ---> 停在某页约 0.8 秒（防抖，避免快速翻页时频繁触发）
        ---> 记阅读：mark_read 写 page_reads（“停下来就算读过”，并更新 max_read_page）
        ---> 记视图：save_view_state（当前页 + 单/双页，供下次续读）
        ---> 后台线程，对当前可见页判断 has_page（是否已索引？）
              ---> 没索引：切块 ---> 向量化 ---> 存进 Chroma（工具栏显示「索引中…」）
              ---> 已索引：跳过（不重复嵌入）
```

关窗时再存一次最终视图，保证续读准确。

---

## 三、对话时

```
输入问题、回车发送
  ---> 锁住“发送”、显示「思考中…」（输入框仍可继续打字，等回复完才能再发）
  ---> 后台线程：
        ---> 检索：问题向量化 ---> 在 Chroma 里按 companion 过滤
                   （document_id 相同 且 page_number <= max_read_page）取最相关的 top-k chunk
        ---> 拼 prompt：system 提示  +  对话历史  +  [相关资料(检索到的chunk) + 当前页内容 + 我的问题]
        ---> 流式调用 LLM ---> 一段段显示到 AI 气泡（引用资料时标注「见第 N 页」）
  ---> 回复结束：
        ---> 把（纯问题 + 完整回答）存进对话记忆
        ---> 当前页内容、检索到的 chunk 都【不进历史】（只是本轮临时上下文）
        ---> 焦点回到输入框，可直接接着问
```

要点：**对话记忆只长存“问答”**；当前页和检索内容每轮现取现拼、用完即弃，避免上下文越堆越大。
